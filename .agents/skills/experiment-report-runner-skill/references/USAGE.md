# Usage

Use this skill when the codebase already exists, the user already ran experiments, and the next step is to turn existing files in `outputs/` into the final report.

## Typical invocation

```text
Use the experiment-report-runner skill.
The repository is already implemented and the experiments were already run by the user.
Read README.md to identify which experiments the final report should cover.
Then inspect outputs/ and use only the files found there as evidence for metrics, figures, tables, and conclusions.
Write the final report into docs/report/report.tex.
Use the report structure from references/report-template.tex and follow latex-writing-skill when writing the LaTeX.
```

## Expected supporting files

The skill works best when the repository contains or provides:
- `README.md`
- project plan and/or implementation plan
- `outputs/` with experiment artifacts already generated by the user
- configs/logs/scripts/entrypoints that help interpret those artifacts
- `docs/report/report.tex` (optional, may be created or updated)
- a report scaffold in `references`, for example:
  - `references/report-template.tex`
  - `references/report-outline.md`

## Notes

- This skill is for artifact inspection and report writing, not experiment execution.
- It should treat README/project docs as the expected experiment list and `outputs/` as the evidence source.
- The final report must reflect actual results, partial results, missing outputs, or clearly documented ambiguities.
- It should only suggest rerunning experiments as a separate follow-up, not as the default behavior.
