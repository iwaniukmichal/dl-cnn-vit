\begin{abstract}
This report reconstructs the completed study directly from the repository artifacts already stored in~\texttt{outputs/}. The original project plan asked how a convolutional backbone and a vision transformer react to changes in optimization, augmentation, and low-data training on \emph{CINIC-10}. In the executed repository, \emph{EfficientNet-B3} was used instead of the originally planned ResNet-18 as the main CNN reference, while \emph{DeiT-Tiny} remained the transformer baseline. Across three seeds, the pretrained backbones clearly outperformed the custom CNN in Phase~1, with EfficientNet-B3 reaching $91.37 \pm 0.19$ and DeiT-Tiny $88.94 \pm 0.11$ test accuracy. Phase~2 showed that cosine scheduling was the most reliable improvement for both backbones, while EfficientNet-B3 was especially sensitive to an overly large learning rate. Phase~3 found that the repository's \texttt{advanced} augmentation variant, implemented as CutMix-based augmentation, produced the best reduced-data results for both backbones; the \texttt{combined} variant was consistently worse. In Phase~4, standard supervised learning on the reduced $10\%$ subset remained stronger than the comparable closed-set evaluations derived from Prototypical Networks, although few-shot training produced competitive results and informative embeddings. Embedding analysis across all three seeds revealed a useful asymmetry: few-shot DeiT-Tiny formed substantially cleaner clusters than few-shot EfficientNet-B3 despite lower classification accuracy. Ensemble methods did not improve over the best single full-data model. The main conclusion is that architecture-dependent behaviour is real, but in this repository the dominant practical factors were scheduler choice, careful regularization, and the difference between episodic and closed-set evaluation.
\end{abstract}

\section{Research Problem Description}
\label{sec:problem}

The goal of the project was to compare how two different image-classification paradigms behave under the same experimental pipeline: a convolutional network and a vision transformer. The original plan framed this as a comparison between ResNet-18 and DeiT-Tiny, but the implemented repository and the completed outputs replaced ResNet-18 with EfficientNet-B3. This report therefore follows the executed study rather than the hypothetical plan and treats EfficientNet-B3 as the CNN reference model throughout.

The study was organized into four main phases. Phase~1 established baseline performance for three models: a custom CNN trained from scratch, EfficientNet-B3, and DeiT-Tiny. Phase~2 performed a fixed hyperparameter search for the two pretrained backbones. Phase~3 tested four augmentation strategies under the tuned Phase~2 hyperparameters. Phase~4 extended the comparison to low-data learning through Prototypical Networks, reduced-data supervised baselines, embedding-space analysis, and downstream ensemble-style evaluations.

The central research questions were:

\begin{enumerate}[nosep]
    \item Which backbone is strongest under default supervised training on \emph{CINIC-10}?
    \item Which optimization and regularization settings are most helpful for the CNN and for the transformer?
    \item Do augmentation effects differ across architectures, and if so, which strategy is the most reliable?
    \item Under low-data conditions, does few-shot training outperform standard supervised training, and how do the learned embedding spaces differ?
\end{enumerate}

This report uses the project plan to preserve the intended structure, the README to reconstruct the workflow, and the files already present in~\texttt{outputs/} as the sole evidence base for all quantitative claims. No experiments were rerun while preparing the report.

\section{Theoretical Introduction and Literature Review}
\label{sec:theory}

% TODO: add bibliography entries for CINIC-10, EfficientNet, DeiT, and Prototypical Networks if the report is extended into a citation-backed paper.

The repository compares two families of representation learners with different inductive biases. Convolutional neural networks build locality and translation-equivariance directly into the architecture, which usually makes them efficient and stable on moderate-size image datasets. Vision transformers replace local convolutional filters with token-wise self-attention, which gives them more flexible global interactions but often makes them more dependent on optimization details, regularization, and augmentation quality.

These architectural differences matter most when the dataset is not extremely large. \emph{CINIC-10} is larger than CIFAR-10 but still far smaller than the corpora on which modern transformers are usually pretrained or distilled. Under such conditions, transfer learning, learning-rate choice, scheduler design, stochastic regularization, and augmentation policy can change the relative standing of the models more than the backbone family alone. This is exactly why the project plan emphasized a phased comparison instead of a single benchmark table.

The few-shot part of the project adds a second perspective. In standard supervised classification, the model is optimized directly for closed-set prediction over the ten dataset classes. In a Prototypical Network, the encoder instead learns an embedding space in which class means act as prototypes and new samples are classified by distance to those prototypes. This makes episodic training attractive in low-data settings, but it also changes the meaning of the evaluation metric: a $5$-way episodic score is not directly comparable to a standard $10$-class test accuracy. The repository therefore includes two bridge evaluations after few-shot training: a $10$-way episodic test and a fixed-prototype classifier built from sampled training embeddings.

Embedding analysis provides a geometric view of the same problem. The silhouette score measures how well each point matches its own class cluster relative to other clusters, so higher values are better. The Davies--Bouldin index measures the ratio of within-cluster scatter to between-cluster separation, so lower values are better. The inter-class / intra-class distance ratio summarizes how strongly class centers separate relative to class spread, so higher values are better. These metrics do not replace accuracy, but they help explain whether a model succeeds because it learns a globally well-structured feature space or because it is merely good at the final decision rule used during evaluation.

The repository does not ship a bibliography or citation infrastructure for a formal literature review, so this section remains a concise theoretical framing rather than a source-heavy survey. The rest of the report therefore focuses on evidence recoverable from the local experiment artifacts.

\subsection{Phase-specific methodology}
\label{subsec:methodology}

Phase~1 used full-data supervised training. The custom CNN was trained for $15$ epochs with AdamW, learning rate $10^{-3}$, weight decay $5 \times 10^{-4}$, batch size $128$, no scheduler, and \texttt{baseline} augmentation. The two pretrained models were fine-tuned for $5$ epochs with AdamW and no scheduler: EfficientNet-B3 used learning rate $3 \times 10^{-4}$, weight decay $10^{-4}$, drop-path $0.0$, batch size $64$, input size $300$, and \texttt{baseline} augmentation, while DeiT-Tiny used learning rate $5 \times 10^{-5}$, weight decay $0.05$, drop-path $0.0$, batch size $64$, input size $224$, and the same \texttt{baseline} augmentation.

Phase~2 performed a $2^4$ grid search over learning rate, scheduler, drop-path rate, and weight decay on a deterministic class-balanced $5\%$ subset. EfficientNet-B3 searched over learning rates $\{3 \times 10^{-4}, 10^{-3}\}$, schedulers \{\texttt{none}, \texttt{cosine}\}, drop-path rates $\{0.0, 0.1\}$, and weight decay $\{10^{-4}, 5 \times 10^{-4}\}$. DeiT-Tiny searched over learning rates $\{5 \times 10^{-5}, 2 \times 10^{-4}\}$, the same scheduler options, drop-path rates $\{0.0, 0.1\}$, and weight decay $\{0.05, 0.1\}$. Search selection was based on mean validation accuracy across the three seeds.

Phase~3 reused the winning Phase~2 hyperparameters for each pretrained backbone and compared the four augmentation strategies from Table~\ref{tab:augmentations} on the $5\%$ subset.

Phase~4 used few-shot learning with Prototypical Networks on a deterministic $10\%$ subset, using $5$-way, $5$-shot, and $10$-query episodes, together with $200$ training episodes, $100$ validation episodes, and $200$ test episodes per run. Each backbone was trained with the same four augmentation variants as in Phase~3 and reused the winning Phase~2 hyperparameters. Because the standard ProtoNet $5$-way episodic test accuracy is not directly comparable to models evaluated over all $10$ classes, two additional evaluations were performed for every saved checkpoint: a $10$-way episodic test and a prototype-based classifier built from $64$ sampled training examples per class. In the latter setting, each class prototype was computed from $64$ random training examples. Since test-set labels are typically unavailable during inference, this provides a fairer comparison with standard supervised models, as no information from the test set is used during inference.

Phase~4 also included reduced-data and full-data supervised comparisons. The executed configs reused the winning Phase~2 hyperparameters, Phase~3 augmentation strategy and a $10\%$ subset for the reduced-data runs.

Finally, the repository contains four post-hoc embedding-analysis jobs and three downstream ensemble jobs. The embedding-analysis configs read the Phase~4 Protonet winning checkpoints and the Phase~4 full-data supervised checkpoints, then extract $100$ train and $100$ test embeddings per class for all three seeds. The ensemble phase evaluates soft voting by averaging the logits of the full-data EfficientNet-B3 and DeiT-Tiny checkpoints, evaluates stacking by concatenating their features and fitting a logistic-regression classifier on top, and trains logistic-regression probes on frozen embeddings from the selected Protonet checkpoints.

Table~\ref{tab:phase2-best} shows the selected Phase~2 configurations. Cosine scheduling is the clearest common pattern: averaged across the whole grid, it raises mean validation accuracy from $80.95\%$ to $84.18\%$ for EfficientNet-B3 and from $78.98\%$ to $80.85\%$ for DeiT-Tiny. This makes the scheduler the most robust backbone-agnostic improvement discovered in the search.

The second strong trend is learning-rate sensitivity. EfficientNet-B3 clearly prefers the smaller step size: averaging over the entire grid, $3 \times 10^{-4}$ yields $84.32\%$ mean validation accuracy, whereas $10^{-3}$ drops to $80.81\%$. Drop-path and weight decay matter less than scheduler choice in the aggregated search results. For EfficientNet-B3 they provide small but positive nudges toward the selected setting. For DeiT-Tiny their marginal effects are almost neutral in the factor averages, which again suggests that most of the benefit comes from the right optimizer-scheduler combination rather than from isolated regularization terms.

The same conclusion holds for DeiT-Tiny: the best Protonet best prototype-based classifier $82.61\%$, both below the reduced supervised DeiT-Tiny result of $84.64\%$.

The gap widens further on the full dataset. Relative to the Phase~1 baseline, the full-data \texttt{advanced} supervised pipeline improves EfficientNet-B3 from $91.37\%$ to $93.99\%$ and DeiT-Tiny from $88.94\%$ to $90.52\%$. This indicates that, in the current implementation, low-data few-shot training did not replace conventional supervised learning. However, the $10\%$ subset still corresponds to $9000$ training examples per epoch, which is relatively large; few-shot learning may prove more beneficial in even lower-data regimes.

The strongest general pattern in Table~\ref{tab:embedding-results} is that full-data supervised models have much cleaner embeddings than few-shot models. Both backbones improve substantially on~all three geometry metrics when moving from the few-shot checkpoint lineage to the full-data supervised checkpoint lineage.

The more subtle result is a~cross-architecture disagreement between geometry and accuracy. EfficientNet-B3 is the stronger classifier in~every accuracy table, yet few-shot DeiT-Tiny produces much cleaner embedding geometry: its silhouette score is nearly twice as~high as~that of~few-shot EfficientNet-B3 ($0.213$ versus $0.117$), its Davies--Bouldin index is much lower ($1.667$ versus $2.461$), and its distance ratio is far larger ($1.779$ versus $1.125$). This implies that better representation geometry does not automatically translate into higher closed-set accuracy across backbones. Under the current evaluation pipeline, EfficientNet-B3 extracts more useful decision features for classification, while DeiT-Tiny organizes the feature space more coherently in~the few-shot setting.

Figures~\ref{fig:pca-train} and~\ref{fig:pca-test} show the corresponding PCA projections. In~the first row, corresponding to~the few-shot setting, DeiT-Tiny produces more clearly defined clusters than EfficientNet-B3. In~the supervised setting shown in~the second row, the EfficientNet-B3 clusters are more clearly separable, which is consistent with results in tab.~\ref{tab:embedding-results}.

However, Figures~\ref{fig:tsne-train} and~\ref{fig:tsne-test} in~Appendix~\ref{sec:appendix-plots} do not visualise this phenomenon as~clearly in~the t-SNE projections for seed~13. Both backbones produce well-separated clusters, although the EfficientNet-B3 embedding space appears slightly more compact.

Figures~\ref{fig:heatmap-train} and~\ref{fig:heatmap-test} display the centroid distance matrices. Each cell shows the Euclidean distance between the class centroids in~the embedding space. The smallest off-diagonal values reveal which class pairs are hardest to~separate: for EfficientNet-B3 the closest pair is automobile--truck in~both training regimes, while for DeiT-Tiny it~is cat--dog. In~all four settings the semantically similar pairs stand out, confirming the conclusions from the visualisations.

The ensemble follow-up in Table~\ref{tab:ensemble-results} does not show additive gains over the best single model. Soft voting (average of the EfficientNet-B3 and DeiT-Tiny logits) almost matches the full-data supervised EfficientNet-B3 result but remains slightly lower ($93.88\%$ versus $93.99\%$). Stacking (feature concatenation + logistic regression) is clearly worse than both soft voting and the best single backbone. In other words, the transformer does not contribute enough complementary error correction to improve on the stronger CNN checkpoint lineage.

The Protonet linear probes are more encouraging, but only modestly so. For EfficientNet-B3 the linear probe improves on the fixed-prototype evaluation from $86.99\%$ to $87.19\%$, and for DeiT-Tiny from $82.61\%$ to $83.04\%$. That means the frozen few-shot embeddings retain useful class information beyond simple nearest-prototype classification. Still, the gains are not large enough to catch the reduced supervised baselines.

\section{Conclusions and Further Research}
\label{sec:conclusions}

The executed study leads to five clear conclusions.

\begin{enumerate}[nosep]
    \item EfficientNet-B3 is the strongest classifier in this repository. It beats DeiT-Tiny in the Phase~1 baseline, the Phase~4 reduced supervised comparison, the Phase~4 full-data comparison, and all few-shot accuracy evaluations.
    \item Cosine scheduling is the most reliable tuning decision for both backbones. EfficientNet-B3 also benefits strongly from keeping the learning rate conservative, whereas DeiT-Tiny shows a more interaction-heavy search surface.
    \item The repository's \texttt{advanced} augmentation, implemented around CutMix, is the best reduced-data supervised choice for both backbones. The \texttt{combined} policy is consistently harmful, which suggests that the current recipe becomes too aggressive in the short reduced-data regime.
    \item Few-shot learning is useful but does not outperform standard supervised learning under comparable closed-set evaluation. On the reduced $10\%$ subset, the best Protonet-derived classifiers remain below the reduced supervised models for both backbones.
    \item Embedding geometry and classification accuracy are related but not identical. Few-shot DeiT-Tiny learns cleaner clusters than few-shot EfficientNet-B3, yet EfficientNet-B3 still classifies better.
\end{enumerate}

The study also has two important limitations. The first is architectural: the final repository substituted EfficientNet-B3 for the originally planned ResNet-18, so the conclusions are about ``CNN versus ViT'' in a broader sense, not specifically about ResNet-18. The second is procedural: the Phase~4 supervised, embedding-analysis, and ensemble follow-up steps are tied to the \texttt{advanced} augmentation lineage rather than to a dynamically selected best Phase~3 lineage.

The most useful next steps are therefore concrete. Reintroducing a true ResNet-18 baseline would restore full consistency with the original plan. Expanding the augmentation space beyond the current CutMix-centric \texttt{advanced} and \texttt{combined} variants would test whether the transformer benefits even more from architecture-specific regularization. Finally, repeating the Phase~4 supervised and downstream ensemble steps with the empirically best Phase~3 lineage, rather than with the fixed \texttt{advanced} lineage, would clarify whether the current full-data reference is merely good or truly optimal within this repository.

\section{Application Instructions}
\label{sec:application}

The repository is operated entirely through command-line entrypoints and YAML configs. Reproduction therefore consists of recreating the environment, placing \emph{CINIC-10} in the expected directory layout, rerunning the phase-specific configs for the planned seed set $\{13, 37, 73\}$, and aggregating the resulting seed directories.

\subsection{Environment and data}
\label{subsec:environment}

Installation:

\begin{verbatim}
python -m venv .venv
source .venv/bin/activate
pip install -e .[dev]
\end{verbatim}

The dataset must be available under:

\begin{verbatim}
data/cinic10/
  train/
  valid/
  test/
\end{verbatim}

with standard \texttt{ImageFolder} class subdirectories in each split.

\subsection{CLI pattern}
\label{subsec:cli-pattern}

Every job follows the same template:

\begin{verbatim}
python -m archdyn.cli.<command> --config <path-to-yaml> --seed <seed>
\end{verbatim}

The main commands are:

\begin{itemize}[nosep]
    \item \texttt{train} for standard supervised runs,
    \item \texttt{search} for Phase~2 hyperparameter search,
    \item \texttt{fewshot} for Protonet training,
    \item \texttt{fewshot\_eval} for the additional $10$-way episodic evaluation,
    \item \texttt{fewshot\_prototype\_eval} for the fixed-prototype closed-set evaluation,
    \item \texttt{analyze\_embeddings} for post-hoc embedding extraction,
    \item \texttt{ensemble} for soft voting via logits averaging, stacking via feature concatenation with logistic regression, and Protonet linear probes,
    \item \texttt{aggregate} for mean/std summaries across finished seeds.
\end{itemize}

\subsection{Minimal reproduction workflow}
\label{subsec:reproduction}

The README provides the full command list and instructions to reproduce results under \textit{End-to-end experiments} section.

\begin{enumerate}[nosep]
    \item Run all Phase~1 configs from \texttt{configs/phase1/} for seeds~13, 37, and~73, then aggregate \texttt{outputs/phase1/}.
    \item Run the two Phase~2 search configs from \texttt{configs/phase2/}, aggregate the search outputs, and propagate the winning hyperparameters into the later YAMLs.
    \item Run all eight Phase~3 configs from \texttt{configs/phase3/}, then aggregate \texttt{outputs/phase3/}.
    \item Run all eight Protonet configs from \texttt{configs/phase4/}, then run the extra \texttt{fewshot\_eval} and \texttt{fewshot\_prototype\_eval} steps for each saved checkpoint.
    \item Run the four Phase~4 supervised comparison configs and aggregate \texttt{outputs/phase4/}.
    \item Run the four embedding-analysis configs from \texttt{configs/analysis/}.
    \item Run the three ensemble configs from \texttt{configs/ensembles/} and aggregate \texttt{outputs/ensembles/}.
\end{enumerate}


\subsection{Where to inspect results}
\label{subsec:outputs}

All artifacts are written under:

\begin{verbatim}
outputs/<phase>/<experiment_name>/seed_<seed>/
\end{verbatim}

The most important files are:

\begin{itemize}[nosep]
    \item \texttt{test\_metrics.json} for supervised and training-time few-shot metrics,
    \item \texttt{aggregate/metrics\_mean\_std.json} for multi-seed summaries,
    \item \texttt{aggregate/search\_results\_aggregated.csv} and \texttt{best\_search\_result.json} for Phase~2 search analysis,
    \item \texttt{episodic\_eval\_test\_nway\_10.json} and \texttt{prototype\_eval\_train64\_test.json} for the two closed-set Protonet follow-ups,
    \item \texttt{test\_embedding\_metrics.csv}, \texttt{test\_metrics.json}, and the \texttt{plots/} directory for embedding analysis,
    \item \texttt{soft\_voting\_metrics.json}, \texttt{stacking\_metrics.json}, and \\ \texttt{protonet\_logreg\_metrics.json} for the ensemble phase.
\end{itemize}
